在配置 Clash、Sing-box 或 Shadowrocket 等代理软件时，我们经常需要将机场提供的原生订阅格式转换为特定客户端支持的格式。这便需要用到**订阅转换**服务。

然而，频繁使用网络上公开的免费第三方订阅转换接口，其实潜藏着极大的安全隐患。

---

## 1. 为什么使用公共订阅转换不安全？

订阅转换的基本原理是：将您输入的原始机场订阅链接发送给后端服务器，服务器下载该配置、解析节点数据、转换格式后重新打包返回给您。

在这个过程中，**后端服务器的拥有者可以直接获取您的原始订阅链接**。如果遇到恶意节点站长或服务器日志泄露，可能会导致以下风险：
*   **流量被盗用**：他人可以直接使用您的订阅节点消耗您的机场流量。
*   **隐私泄露**：机场链接通常绑定了您的账号信息，部分机场订阅接口还会直接暴露用户的邮箱或注册 IP。
*   **节点劫持**：恶意中转站有可能在返回的配置中夹带他们自己的嗅探节点，从而监控您的明文上网行为。

---

## 2. 最佳防范方案：本地自建订阅转换

如果您注重隐私安全，最彻底的解决办法是使用 **Docker** 在本地电脑（或自建的境外私有 VPS）上搭建一个专属的订阅转换服务。

### 第一步：准备 Docker 环境
确保您的电脑上已经安装并运行了 **Docker Desktop**。

### 第二步：一键运行 Subconverter 容器
在终端（CMD 或 PowerShell）中执行以下命令，即可拉取并运行开源的 `subconverter` 镜像：

```bash
docker run -d --name subconverter -p 25500:25500 tindy2013/subconverter:latest
```

*该命令会把服务运行在本地的 `25500` 端口上，且不会有任何外部流量流出，100% 保证本地安全。*

### 第三步：自建前端页面 (可选)
如果您觉得命令行拼接 URL 太麻烦，可以配合部署前端界面 `sub-web`，或者直接使用可信赖的开源静态前端（如 `sub.xeton.dev`），但在前端设置中的“后端地址”一栏，必须填入您本地的地址：
`http://127.0.0.1:25500`

---

## 3. 常见安全原则

1.  **绝不使用未知小站提供的订阅转换**。
2.  **善用密码/Token 保护**：在使用自建后端时，可以在 `pref.ini` 配置文件中添加 `api_access_token`，只有携带 Token 的请求才允许进行转换，避免被其他人扫描并滥用您的后端。
